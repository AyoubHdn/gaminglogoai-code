## G. Platform-extension readiness verdict

- Can I add a new platform by just adding a new dimension preset, without touching the prompt? No - there is no pixel-dimension preset system today, only freeform aspect ratios ([gaming-logo-maker.tsx](<C:/Users/ULTRAPC/Documents/GitHub/SaaS/gaminglogoai-code/src/pages/gaming-logo-maker.tsx:68>), [generate.ts](<C:/Users/ULTRAPC/Documents/GitHub/SaaS/gaminglogoai-code/src/server/api/routers/generate.ts:62>)).
- Does the prompt need to change per platform? Usually no - logo prompts are generic enough, but safe-area/platform-cropping guidance would still need a platform layer outside the prompt ([gaming-logo-maker.tsx](<C:/Users/ULTRAPC/Documents/GitHub/SaaS/gaminglogoai-code/src/pages/gaming-logo-maker.tsx:168>)).
- Does the UI need a platform selector, or is platform inferred from the URL? A platform selector or preset picker is the safer approach, because this page is intentionally universal and currently only exposes shape choices ([gaming-logo-maker.tsx](<C:/Users/ULTRAPC/Documents/GitHub/SaaS/gaminglogoai-code/src/pages/gaming-logo-maker.tsx:421>)).
- Are there platform-specific visual conventions the AI does not know about that need injection? Yes - banner-safe areas, avatar cropping margins, and icon corner handling are not modeled anywhere in this tool ([generate.ts](<C:/Users/ULTRAPC/Documents/GitHub/SaaS/gaminglogoai-code/src/server/api/routers/generate.ts:57>)).
