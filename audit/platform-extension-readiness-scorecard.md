## Biggest architectural issue to fix before adding platforms

The codebase has no central platform-definition layer. Dimensions, copy, storage keys, prompt wording, style catalogs, and even upload folders are scattered across page files, style data files, and route handlers instead of being modeled once and injected everywhere ([gaming-logo-maker.tsx](<C:/Users/ULTRAPC/Documents/GitHub/SaaS/gaminglogoai-code/src/pages/gaming-logo-maker.tsx:68>), [twitchBannerStyles.ts](<C:/Users/ULTRAPC/Documents/GitHub/SaaS/gaminglogoai-code/src/data/twitchBannerStyles.ts:75>), [s3.ts](<C:/Users/ULTRAPC/Documents/GitHub/SaaS/gaminglogoai-code/src/server/api/routers/s3.ts:34>), [emoteGenerator.ts](<C:/Users/ULTRAPC/Documents/GitHub/SaaS/gaminglogoai-code/src/server/api/routers/emoteGenerator.ts:142>)). If you add YouTube/Kick/TikTok/Discord without first centralizing this, the same platform assumption will have to be updated in half a dozen places per tool.

## Biggest code-sharing opportunity being missed

The three Twitch graphic generators are nearly the same renderer: load a background, resize with `sharp`, build SVG text, render via Puppeteer, create an `Icon`, upload to S3, and return a URL ([twitchBanner.ts](<C:/Users/ULTRAPC/Documents/GitHub/SaaS/gaminglogoai-code/src/server/api/routers/twitchBanner.ts:70>), [twitchPanel.ts](<C:/Users/ULTRAPC/Documents/GitHub/SaaS/gaminglogoai-code/src/server/api/routers/twitchPanel.ts:87>), [twitchStreamScreen.ts](<C:/Users/ULTRAPC/Documents/GitHub/SaaS/gaminglogoai-code/src/server/api/routers/twitchStreamScreen.ts:74>)). A shared "templated graphic renderer" plus a shared "credit + icon + S3 upload" wrapper would remove the heaviest duplication and make platform expansion much safer.

## Tool most likely cheaper to rewrite than extend

`/emote-generator` is the strongest rewrite candidate. The route schema hardcodes `platform: "twitch"`, the prompt hardcodes "Create a Twitch emote", example assets live under `/twitch/emotes/...`, and the selectable expressions include a Twitch-only culture bucket ([emoteGenerator.ts](<C:/Users/ULTRAPC/Documents/GitHub/SaaS/gaminglogoai-code/src/server/api/routers/emoteGenerator.ts:107>), [emoteGenerator.ts](<C:/Users/ULTRAPC/Documents/GitHub/SaaS/gaminglogoai-code/src/server/api/routers/emoteGenerator.ts:142>), [emote-generator.tsx](<C:/Users/ULTRAPC/Documents/GitHub/SaaS/gaminglogoai-code/src/pages/emote-generator.tsx:213>), [emotes.ts](<C:/Users/ULTRAPC/Documents/GitHub/SaaS/gaminglogoai-code/src/data/emotes.ts:40>)). A generic "reaction sticker / emote pack" abstraction would probably be cheaper than layering more platforms onto the current Twitch-first shape.
